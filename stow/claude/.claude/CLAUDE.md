# personal preferences

## style: lowercase by default

prose i write or you write — chat replies, sentence starts in markdown, headers in new docs, commit messages: lowercase.

proper nouns and acronyms keep their case: Sam, RDQ, GetFPV, FPV, GPS, U.S., ArduPilot, Bambu Lab, EMAX, etc.

code identifiers, file paths, SKUs, manufacturer part numbers, URLs, command-line flags: keep canonical casing.

when in doubt, match how i'm writing. i write lowercase except the carve-outs above.
